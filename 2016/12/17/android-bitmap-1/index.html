android-bitmap(1)

发表于 2016-12-17

本文涉及：图片来源、图片处理、

图片来源：BitmapFactory

在android中，提供了BitmapFactory类，用作于从不同数据源解析、创建提供Bitmap对象。（Creates Bitmap objects from various sources , including files, strems, and byte-arrays）

提供的方法

static Bitmap decodeByteArray(byte[] date,int offset,int length);
static Bitmap decodeByteArray(byte[] date,int offset,int length,BitmapFactory.Option opts);
static Bitmap decodeFile(String pathName);
static Bitmap decodeFile(String pathName,BitmapFactory.Option opts);
static Bitmap decodeFileDescriptor(FileDescriptor fd) ;
static Bitmap decodeFileDescriptor(FileDescriptor fd, Rect outPadding, BitmapFactory.Options opts);
static Bitmap decodeResource(Resources res, int id, BitmapFactory.Options opts) ;
static Bitmap decodeResource(Resources res, int id) ;
static Bitmap decodeResourceStream(Resources res, TypedValue value, InputStream is, Rect pad, BitmapFactory.Options opts);
static Bitmap decodeStream(InputStream is);
static Bitmap decodeStream(InputStream is, Rect outPadding, BitmapFactory.Options opts);

通过BitmapFactory.Option对产生的bitmap进行限定

1、首先获得一个BitmapFactory.Option对象

BitmapFactory.Option opts=new BitmapFactory.Option();

2、对opts的属性进行设置

//例如：
opts.inJustDecodeBounds=true;

3、在获得bitmap对象时将参数opts传进去

Bitmap bitmap=BitmapFactory.decodeStream(XXXX,XXXX,opts);

常用方法

inJustDecodeBounds: 如果值为true，那么将不返回实际的bitmap，也不给其分配内存空间，这样就避免了内存的溢出。但是允许我们查询图片的信息，这其中就包括了图片的大小信息。

inSampleSize: 可以利用它，实现缩放。如果设置为>1，要求解码器解码出原始图片的一个子样本，返回一个较小的bitmap，以节省空间。例如，inSampleSize=2，那么得到的缩略图宽和高都是原始图片的1/2，图片的大小是原始图片的1/4.对于任意值<=1都当成1来处理。

inPreferredConfig：设置解码方法，解码方式不同在内存中所占用的大小也不相同。一共有四种解码方法：ARGB_8888、ARGB_565、ARGB_4444、ALPHA_8.后面的数字位数代表的是通道数，数字大小是指该通道占的位数，所有通道占的位数和表示以该方式解码后的bitmap一个像素所占的内存大小。
